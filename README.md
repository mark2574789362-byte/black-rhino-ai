# Black Rhino AI 商品运营诊断助手

> 面向商品运营场景的 AI 诊断原型：把商品定位、Listing 优化、Bundle 套餐设计、SEO 内容方向和数据验证指标规划，拆解成可落地、可解释、有数据边界的运营分析流程。

- **在线 Demo**：https://black-rhino-ai.pages.dev/
- **GitHub**：https://github.com/mark2574789362-byte/black-rhino-ai

## 项目简介

Black Rhino AI 是一个基于真实电商商品运营场景设计的 AI 产品原型。用户可以输入公开商品信息或手动整理的产品卡片，系统会生成结构化诊断结果，帮助运营人员快速完成初步判断。

这个项目的重点不是简单“接入一个大模型”，而是把运营分析流程拆成清晰、可控的产品能力：

- 商品定位分析
- Listing 诊断与优化建议
- 标题、卖点、描述检查
- Bundle 套餐组合建议
- SEO 关键词与内容方向
- 后续数据验证指标规划
- 数据充分性评分与风险提示

---

## 核心亮点

### 1. 面向真实业务场景，而不是 AI 套壳

项目围绕商品运营人员的实际工作流设计：先判断商品定位，再检查 Listing 表达，进一步给出套餐、SEO 和验证指标建议。输出结果不是泛泛聊天，而是可用于运营分析的结构化内容。

### 2. 有明确的数据边界意识

本工具**不访问 Black Rhino 后台数据**，仅基于公开商品信息或用户手动输入内容进行诊断。

因此系统不会在缺少数据时强行推断以下内容：

- 真实销售额 / GMV
- 毛利率 / 利润率
- 库存周转 / 断货率
- 广告 ROI / 投产比
- 真实 SKU ABC 分类

如果信息不足，系统会优先提示缺失字段，而不是让 AI 编造结论。

### 3. 规则层守门，提升 AI 输出可信度

在调用大模型前增加规则校验：

- 输入信息不足时，不直接调用 API
- 明确提示还需要补充哪些字段
- 输出结果始终包含 Data Sufficiency Score
- 对没有数据支撑的判断进行限制

这样可以降低 AI 幻觉，让诊断结果更适合真实业务场景。

### 4. 完整部署上线

项目已完成前端开发、接口封装、大模型接入和线上部署，可以直接通过 Demo 体验。

## 功能模块

| 模块 | 说明 |
| --- | --- |
| 商品定位分析 | 判断商品面向的人群、核心卖点和潜在使用场景 |
| Listing 诊断 | 分析标题、卖点、描述是否清晰，是否突出转化要素 |
| Bundle 套餐建议 | 基于商品属性提出可能的组合销售方向 |
| SEO 内容方向 | 给出关键词、内容角度和页面优化方向 |
| 数据验证指标 | 规划后续需要观察的业务指标，如转化率、点击率、复购等 |
| 数据充分性评分 | 告诉用户当前输入信息能支持哪些分析，哪些不能判断 |

## 技术架构

```text
React 前端（Vite + TypeScript + Tailwind CSS）
        ↓ fetch /api/analyze
Cloudflare Pages Function
        ↓ MINIMAX_API_KEY env
MiniMax 大模型 API
        ↓
结构化 JSON 结果
        ↓
前端渲染诊断报告
```
